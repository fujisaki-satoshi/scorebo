このモックでカバーする実装範囲(承認済みスコープ)

  • #1 Web Share API ボタン: モーダル最上部の「みんなに送る」緑ボタン。navigator.share({ title, text, url, files }) を呼ぶ。canShare({ files }) が false の端末は url-only にフォールバック。Web Share 非対応端末(主にデスクトップ)ではこのボタン自体を非表示、LINE が暫定主CTA に昇格。
  • #2 スコア画像の自動生成: モーダル上部のプレビューがそのまま ImageResponse で生成される PNG(1200x630)。Web Share / OG 両用。レイアウトは: 緑ヘッダ(ロゴ・日付・場所) → スポーツ表示 → 対戦+大スコア → イニング表 → 下部バー(URL・QR・無料訴求)。
  • #3 OG 画像の動的化: app/games/[id]/opengraph-image.tsx を新規作成し、#2 と同じテンプレで動的生成。SNS で URL を貼ると試合結果プレビューに。
  • #4 コピー強化: コピー時のテキストが URL のみ → 「{team_top} vs {team_bottom} のスコア\n{date}\n{url}」に。LINE の ?text= も同様。
  • #8 Powered by チップ: スコア画面の bottom-right に小さく「スコアボで作成 — 無料」+ ロゴマーク。クリックで / (LP) に遷移。広告に見えない控えめなトーン。

共有モーダルの構造変更点(現状との差分)

  • 従来の「試合カード(対戦+メタ)」を スコア画像プレビュー に置き換え(ユーザーが何を送るか可視化)
  • 主CTA「LINE で送る」(line-green) を 「みんなに送る」(brand-green) に変更。LINE は「個別に送る」セクションの最初に降格
  • 「個別に送る」セクションに LINE / URL コピー を集約
  • QR コードは下部「またはこの場で見せて読み取ってもらう」セクションに残す(対面用途)
  • 注釈文言は変更なし

Analytics(既存に追加するイベント)

  • qr_share_native — Web Share API 経由で送られたとき(成否は判定不可)
  • qr_share_image_loaded(任意) — モーダル表示時に画像生成が成功したか
  • 既存の qr_share_opened / qr_share_line / qr_share_copy は維持

本実装時の懸念・確認点

  • 画像生成のキャッシュ戦略: スコアが更新されるたびに画像も古くなる。Vercel の ImageResponse はキャッシュ可能だが、 updated_at をクエリにつけて invalidate する設計が要る。
  • モバイル Web Share API 制約: iOS Safari は files 共有対応(15.4+)。Chrome Android は対応済。古い端末では url-only fallback。
  • スコア画像のフォント: 既存の opengraph-image.tsx は Satori デフォルトフォントで日本語不可な構造だった。今回は **日本語フォント(Noto Sans JP の数字以外含む) を fetch(googleFontUrl) で読み込んでフォントオプションに渡す** 必要がある。チーム名・場所が日本語なので必須。
  • イニング数の可変対応: 画像内の表は最大何回分まで表示するか?案: 1〜7回までは固定列、それ以上は横スクロール風に縮小 or 「7回以降は省略」とする。決め打ち初期実装は最大9列、それを超えたら 1, 2, 3, ..., last の省略表示。
  • Powered by チップ: 試合作成者(自分)の画面でも出す? 案: 全員に出す(自分にも控えめ表示で違和感小、訪問者には認識効果)。設計判断要。
